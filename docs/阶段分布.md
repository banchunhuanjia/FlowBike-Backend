🛠️ P1 阶段：项目终极完善 (现在 - 12月底)
目标： 写完代码，不再动大架构，让项目“能跑、能扛、有数据”。

第 1 周 (11.25 - 12.01)：工程化扫尾（当前阶段）

任务： 完成 Config 加载（去掉硬编码）、完成 Spdlog 日志替换（去掉 cout）。

产出： 一个看起来很专业的 Server 启动界面，带日志文件。

心态： 这周很轻松，主要是抄作业和改代码，找回手感。

第 2-3 周 (12.02 - 12.15)：攻克 MySQL (最难的一关)

任务： 学习 MySQL C API，写一个 ConnectionPool (数据库连接池)。

业务： 实现“扫码开锁”逻辑：收到请求 -> 从池里拿连接 -> 查库 -> 扣费/记录 -> 放回连接。

心态： 这两周会很痛苦，会遇到 Link 错误、内存泄漏，但这做完了，你就是 C++ 入门了。

第 4 周 (12.16 - 12.22)：接入 Redis

任务： 引入 hiredis。

业务： 缓存用户信息，或者做简单的分布式锁（模拟抢单车）。

心态： Redis 比 MySQL 简单很多，这周是“甜点”。

第 5 周 (12.23 - 12.31)：性能压测 (简历亮点)

任务： 使用 Webbench 或自己写的 Client，把你的服务器压挂。

产出： 一张 QPS 曲线图。比如“优化前 1w QPS，去锁/加连接池后 8w QPS”。

注意： 这是面试官最爱看的东西。

📚 P2 阶段：基础夯实 (1月 - 春节前)
目标： 应对字节跳动的“八股文轰炸”和“手撕代码”。

算法 (每天 1-2 小时)：

不要乱刷，只刷 《LeetCode Hot 100》 和 《剑指 Offer》。

重点：链表、二叉树、TopK 问题（堆）、动态规划（基本款）。

目标： 哪怕背，也要把这 100 道题背下来。

八股文 (每天 2 小时)：

操作系统： 进程vs线程、虚拟内存、死锁、零拷贝 (Zero Copy)。

网络： TCP 三次握手/四次挥手（状态机必考）、HTTP vs HTTPS、IO 多路复用 (Epoll 原理)。

数据库： 索引 B+ 树、事务隔离级别、Redis 持久化。
